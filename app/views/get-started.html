Get started

You need to request access if you want to develop against the API. There are lots of resources available to support you in creating your application.

1. Request access to the API

Email anne.example@wales.nhs.uk to ask for access, and include:

  • your organisation, for example an NHS Wales health board or a 3rd party
  • the reason for your request, for example that you'd like to use the API to retrieve content from the Welsh Data Dictionary

This will help the terminology service team understand the best way to support your request.

2. Understand FHIR API resources

The API is made up of resources that follow the standard FHIR structure of:

  • CodeSystem, also known as ontology or terminology
  • ValueSet
  • ConceptMap

There are standard operations defined for each resource, such as $lookup, $expand and $translate.

One way to understand this structure is to think of the CodeSystem as a book, and the ValueSet as the chapters within that book. The ConceptMap defines the relationship between concepts, including from one CodeSystem to another.

You can find more information about resources on the HL7 website, including examples and detailed descriptions.

3. Read some common use cases

Some of the most common usages of the terminology service API include:

  • creating or referencing a code system
  • looking up a display term for a code
  • recording data using pre-existing codes
  • validating a code
  • mapping data between different terminologies

Read some common example scenarios in FHIR to see further ways that the API can be used.

You may also like to read examples of SNOMED CT clinical use cases to understand how health and care organisations have implemented terminology services.

4. Know where you can get help

Help is available if you need integration support.

For specific technical questions, you may want to join the FHIR community. This FHIR knowledge-sharing website allows you to search through content posted by other users, and to post your own issues and questions.

For more general support with the terminology service, email anne.example@wales.nhs.uk.
