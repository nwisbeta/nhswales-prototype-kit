Get started

1. Requesting access

Information:

You cannot access the API as a 3rd party if you are not sponsored by an NHS Wales health board

Email terminology.service@wales.nhs.uk to request access and include:

  • which health board you work for or are sponsored by
  • how you plan to use the API, for example you'd like to retrieve content from the Welsh Data Dictionary

If you are a 3rd party, it can take up to 5 months to go from your initial request to accessing the API, depending on your requirements. The process for organisations within NHS Wales is normally quicker. See an example timescale for accessing APIs*.

2. Understanding the FHIR API structure

FHIR APIs are created from a standard framework of components called resources. Resources are the building blocks of every request.

Types of FHIR API resources
Resource Definition
CodeSystem A collection of codes also known as ontology or terminology, in this case SNOMED CT
ValueSet A specific subset of codes from the CodeSystem, for example all codes relating to influenza
ConceptMap Defines the relationship between concepts, including from one CodeSystem to another

Each resource is structured in the same way, with:

  • a URL that identifies the resource
  • common metadata
  • an XHTML summary
  • a set of defined data elements
  • an extensibility framework to support variation in healthcare

Read more in HL7's FHIR overview for developers (opens in a new tab), which includes example requests and responses.

3. Making API requests

FHIR APIs have a simple set of interactions like READ and UPDATE that follow the RESTful structure. To actively formulate content inputs and outputs, you use operations.

Operations are structured by pairing the URL from the FHIR endpoint with the name of an operation, which is prefixed by a dollar symbol ($). For example:

GET http://exampleURL.com/path/ResourceType/id/$operation

Frequently used operations include $lookup, $expand and $translate. View the full list of available operations (opens in a new tab).

Expanding a ValueSet

The most common use case for the API is to search and retrieve specific SNOMED CT codes. You can do this using the $expand a ValueSet operation.

The rules of this operation can be broad, like retrieving codes related to influenza. They can also be very specific, like retrieving codes related to influenza that were created on a specific date.

You can see some example $expand requests (opens in a new tab) in action in a Postman environment on the Ontoserver website.

4. Getting support

If you need integration support, help is available.

For specific technical questions, you may want to join the FHIR community (opens in a new tab). This FHIR knowledge-sharing website allows you to search through content posted by other users, and to post your own issues and questions.

For more general help with the terminology service, email terminology.service@wales.nhs.uk for support.
