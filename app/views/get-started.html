Get started

Request access

The environments that you can request access to include:

  • sandbox, to develop a feature that uses the API
  • SIT, for validating the application against integration testing requirements.
  • UAT, for application end users to test the new feature in a non-production environment (if required)
  • production, for pilot and go-live of your application

Anyone can try the APIs in the sandbox to see how they work. However, you must work for NHS Wales or be sponsored by an NHS Wales organisation to get production access.

Get Sandbox access (all users)

Use the sandbox environment to develop a feature that uses the API.

To get access:

  1. Register for an NHS Wales Developer Portal account
  2. Get a subscription key
Get SIT, UAT and production access (health boards and 3rd party partners)

Submit a production access request when you're ready to go live with your application. SIT and UAT access is given first, and used to make sure your application meets our assurance criteria. Production access will be provided once your application passes assurance checks.

To get access:

  1. Fill in and submit a New Service Request to request access to the API.
  2. Your request will be reviewed by the Service Portfolio CAB for approval or rejection.
  3. Submit a ServicePoint call to National Operational Databases (NOD).
  4. Request goes to IRAT SMB where it is approved or denied.
  5. If approved, you're given connection details. If denied, you're given the reason why.
